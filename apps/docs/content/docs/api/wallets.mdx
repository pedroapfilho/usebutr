## Provider

### `AutoWalletManagerProvider`

Drop-in provider that composes EVM + SVM discovery and persists to
`localStorage`.

```tsx
<AutoWalletManagerProvider
  auto={true /* | { evm?, svm?, injected? } */}
  storageKeyPrefix="myapp"
  storage={customWalletStorage}
  onConnect={…}
  onConnectError={…}
  onHydrated={…}
  onDisconnect={…}
  onReset={…}
  onSlowConnect={…}
  onStorageError={…}
  slowConnectThresholdMs={5000}
>
  {children}
</AutoWalletManagerProvider>
```

Props: `AutoWalletManagerProviderProps`. The `on*` / `slowConnectThresholdMs` /
`storage` / `storageKeyPrefix` props pass straight through to
`WalletManagerConfig` — see [lifecycle callbacks](/docs/guides/lifecycle-callbacks).

### `useDiscoveredWallets(): ReadonlyArray<WalletAdapter>`

The live list of adapters announced under the auto provider.

## Discovery primitives

### `discoverWalletAdapters(onAdapter, options?: DiscoverOptions): () => void`

Composed EVM + SVM discovery. `DiscoverOptions = { evm?; svm?; injected? }`.

### `resolveDiscoverOptions(auto: true | false | DiscoverOptions)`

Normalises the `auto` prop. `true` → everything on. Object form is opt-in:
unspecified flags default to `false`; `injected` defaults to `true` only when
`evm` is true. Returns `{ active, evm, injected, svm }`.

### `createDiscoveryBus(onAdapter): DiscoveryBus`

Lower-level multiplexer. Types: `DiscoveryBus`, `DiscoveryPath`.

### `createDiscoveryWalletSource(): WalletSource`

A `WalletSource` (the core discovery seam) backed by composed discovery.

## Chains

- `CHAINS` — `{ evm: Record<…>, svm: Record<…> }`.
- `CHAINS_BY_PLATFORM` — `Record<"evm" | "svm", ChainBase[]>`. Index with
  `wallet.connector.chainPlatform`.
